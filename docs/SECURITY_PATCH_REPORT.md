## Detailed Patch Descriptions

### Patch 1: MQTT TLS Enforcement (mqtt.service.ts)
**Before:** `mqtt://localhost:1883` (plaintext fallback)  
**After:** `mqtts://localhost:8883` (TLS enforced in production)

**Security Impact:** Prevents MITM attacks on MQTT broker connection.

### Patch 2: Credential Obfuscation (credential_manager.cpp)
**Before:** Credentials stored as plaintext strings in NVS  
**After:** XOR encryption using device-specific key derived from MAC address

**Security Impact:** Prevents immediate credential exposure from NVS flash dump.

### Patch 3: MQTT TLS Enabled (docker-compose.yml)
**Before:** `EMQX_MQTT__TLS_ENABLE: "false"`  
**After:** `EMQX_MQTT__TLS_ENABLE: "true"` + TLS cert volume

**Security Impact:** All MQTT traffic encrypted in transit.

### Patch 4: Buffer Overflow Prevention (mqtt_client.cpp)
**Before:** PubSubClient default 256 bytes  
**After:** Increased to 1024 bytes

**Security Impact:** Prevents ESP32 crash from oversized MQTT messages.

### Patch 5: Device List Truncation (main.cpp)
**Before:** Unlimited device iteration  
**After:** Max 50 devices per poll, 30 devices per usage report

**Security Impact:** Prevents heap overflow when router returns many devices.

### Patch 6: Heartbeat Optimization (config.h)
**Before:** `HEARTBEAT_MS = 30000` (30 seconds)  
**After:** `HEARTBEAT_MS = 300000` (300 seconds)

**Security Impact:** Reduces MQTT broker load by 90% (from ~105K to ~10.5K msgs/year per device).
